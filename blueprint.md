# Blueprint: Here Capture

## Overview

Here Capture는 사용자가 지정한 URL의 전체 웹페이지를 캡처하여 고해상도 이미지로 제공하는 웹 애플리케이션입니다. 복잡한 설정이나 브라우저 확장 프로그램 설치 없이, URL을 입력하고 버튼을 클릭하는 것만으로 웹페이지 전체를 손쉽게 저장할 수 있습니다.

## Core Features

-   **URL 기반 전체 페이지 캡처:** 사용자가 입력한 URL의 웹페이지를 처음부터 끝까지 전체 캡처합니다.
-   **고해상도 이미지 생성:** 캡처된 웹페이지를 선명한 고해상도 PNG 이미지로 제공합니다.
-   **간편한 다운로드:** 생성된 이미지를 클릭 한 번으로 사용자의 기기에 다운로드할 수 있습니다.
-   **실시간 처리 및 피드백:** 캡처가 진행되는 동안 사용자에게 로딩 상태를 보여주고, 완료 즉시 결과 이미지를 표시합니다.
-   **지능형 팝업 처리:** 캡처를 방해할 수 있는 쿠키 동의 배너나 광고 팝업 등을 자동으로 처리하여 콘텐츠 중심의 깔끔한 결과물을 제공합니다.

## Design & Style

-   **모던하고 직관적인 UI:** 사용자가 기능을 쉽게 이해하고 사용할 수 있도록 간결하고 현대적인 디자인을 적용했습니다.
-   **반응형 레이아웃:** 데스크톱, 태블릿, 모바일 등 다양한 기기에서 최적의 사용 경험을 제공합니다.
-   **세련된 시각적 요소:**
    -   **폰트:** 가독성이 뛰어난 Noto Sans KR과 Inter 폰트를 사용하여 정보를 명확하게 전달합니다.
    -   **색상:** 신뢰감을 주는 네이비(#2c3e50)와 활기찬 민트(#1abc9c)를 포인트로 사용하여 생동감 있는 인터페이스를 구성합니다.
    -   **효과:** 부드러운 그림자(box-shadow)와 은은한 전환(transition) 효과를 사용하여 고급스러운 느낌을 더했습니다.

## Current Task: 흰 화면 캡처 오류 해결

**목표:** 일부 웹사이트 캡처 시 흰 화면만 표시되는 문제를 해결하고, 어떤 환경에서도 안정적으로 콘텐츠가 렌더링된 후 캡처를 수행하도록 서버 로직을 전면 개선합니다.

**문제 진단:**

-   현재 로직이 페이지의 모든 리소스(이미지, 스크립트, 폰트)가 완전히 로드 및 렌더링되기 전에 캡처를 시도하여, 내용이 없는 흰 화면이 촬영되는 것으로 보입니다. 이는 특히 리소스 로드가 많은 복잡한 사이트에서 발생합니다.

**개선 계획:**

1.  **`server.js` 캡처 로직 재구축:**
    -   **안정적인 실행 환경 구성:** Puppeteer 실행 인수에 `--disable-dev-shm-usage`를 추가하여, 일부 시스템 환경에서 발생할 수 있는 메모리 문제를 예방합니다.
    -   **다단계 로딩 및 대기 전략 도입:**
        1.  **1단계 (DOM 로드):** `page.goto`에서 `waitUntil: 'domcontentloaded'` 옵션을 사용하여 페이지의 기본 구조가 로드될 때까지 기다립니다.
        2.  **2단계 (전체 스크롤):** 페이지 하단까지 반복적으로 스크롤하여 지연 로딩(lazy-loading)되는 모든 이미지를 포함한 전체 콘텐츠를 강제로 불러옵니다. 스크롤 후 높이가 더 이상 변하지 않을 때까지 이 작업을 계속하여 모든 요소가 로드되었음을 확인합니다.
        3.  **3단계 (네트워크 안정화):** 스크롤로 인해 발생한 모든 네트워크 요청(이미지, 데이터 등)이 완료될 때까지 `page.waitForNetworkIdle()`을 사용하여 기다립니다.
        4.  **4단계 (최종 렌더링 대기):** 모든 작업이 끝난 후, CSS 애니메이션이나 최종 스크립트 실행이 완료될 수 있도록 짧은 추가 대기 시간(예: 2초)을 부여합니다.
    -   **안전한 팝업 제거:** 팝업 제거 로직이 페이지의 다른 요소를 실수로 제거하지 않도록, 더 구체적인 CSS 선택자를 사용하고 오류 발생 시에도 전체 프로세스가 중단되지 않도록 예외 처리를 추가합니다.

2.  **Git 배포:**
    -   개선된 `server.js` 파일을 스테이징(`git add .`)합니다.
    -   커밋 메시지와 함께 변경 사항을 커밋(`git commit -m "fix: Resolve white screen capture issue"`)합니다.
    -   `main` 브랜치에 변경 사항을 푸시(`git push origin main`)하여 개선된 버전을 배포합니다.
